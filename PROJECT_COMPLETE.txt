## 🎯 CORE CONCEPTS

### What is IncidentResponseEnv?

An OpenEnv-compliant RL environment where:
- **Agent receives**: Real-time alerts, metrics, logs, service status
- **Agent must do**: Investigate, diagnose root cause, apply fix
- **Agent gets rewarded**: For correct diagnosis (0.40) + correct fix (0.35) + time bonus (0.15) + efficiency (0.10)

### Why It's Interesting

Traditional incidents show *symptoms* (high latency, errors) that mask *root causes* (DB exhaustion, memory leak). A smart agent must:
1. Recognize symptoms (obvious)
2. Find the real cause (hard)
3. Apply the right fix (specific)

### The Three Scenarios

| Task | Difficulty | Scenario | Root Cause | Fix |
|------|-----------|----------|-----------|-----|
| **Easy** | 🟢 | payment-service crashes | Java OOM | Restart |
| **Medium** | 🟡 | API delays, queue backup | DB pool max | Scale pool |
| **Hard** | 🔴 | Intermittent latency spikes | Memory leak | Rolling restart |

## 🔧 TECHNICAL DETAILS

### Stack
- **Language**: Python 3.9+ (Docker 3.11)
- **Web**: FastAPI + Uvicorn
- **Validation**: Pydantic with enums
- **LLM**: OpenAI client → HuggingFace API
- **Container**: Docker + Python 3.11-slim

### API Endpoints
```
POST /reset              Initialize episode (task_id optional)
POST /step               Take action (investigate/diagnose/fix/etc)
GET  /state              Get current observation
GET  /tasks              List all available tasks
GET  /health             Health check
GET  /                   Root info
```

### Reward System
```
Diagnosis Correct:  +0.40
Fix Correct:        +0.35
Time Bonus:         +0.15 × (time_remaining / 300s)
Efficiency Bonus:   +0.10 × (1 - wrong_actions / total)
Wrong Action:       -0.10 (per attempt)

Range: [0.0, 1.0]
Success: ≥ 0.5
```
